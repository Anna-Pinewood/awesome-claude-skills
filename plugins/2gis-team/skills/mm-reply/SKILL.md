# mm-reply — написать сообщение в Mattermost в моём стиле

## Когда применять

Триггеры:

- «ответь <кому>», «напиши <кому/где>», «сформулируй ответ»
- ссылка `https://mm.2gis.one/...`

Не применять для Telegram / Slack / почты — там другие скиллы.

## Что нужно для старта

1. **Куда** писать — URL или имя человека/канала.
2. **Что** сказать — свободный текст с интенцией. Если ответ надо поресёрчить, и у тебя есть возможность – поресёрчь.
3. **Тон** — опционально.

Если не хватает (1) или (2) — задай **один атомарный вопрос** до запуска любых скриптов.

## Workflow

По ходу работы пиши пользователю короткие апдейты — что делаешь сейчас.

### Шаг 1. Прочитать контекст

Если вход — **permalink в треде** (`/pl/<id>`):

```bash
# одно сообщение (default) — метаданные + тело
uv run --no-project python ${CLAUDE_PLUGIN_ROOT}/scripts/mm_fetch.py '<permalink>'

# root + все ответы в треде — нужно, чтобы ответить именно в этом треде
uv run --no-project python ${CLAUDE_PLUGIN_ROOT}/scripts/mm_fetch.py '<permalink>' --mode thread
```

`--mode thread` сам поднимется к root, если permalink ведёт на reply.

Если вход — **URL канала / ЛС** или название человека:

```bash
uv run --no-project python ${CLAUDE_PLUGIN_ROOT}/scripts/mm_read.py '<url>' --limit 10
```

Дефолты: для ЛС `--mode all`, для канала `--mode root`. Это правильно в большинстве случаев.

Из вывода зафиксируй: участников, язык, стиль (формальный / разговорный / эмодзи / markdown), последние темы.

### Шаг 2. Хватает ли моих сообщений?

Посчитай в выводе шага 1: **сколько сообщений от Ольги Липиной** (`@o.lipina` / `Ольга Липина`) и **сумма слов**.

- ≥ 3 сообщений И ≥ 70 слов → стиля достаточно, переходи к шагу 3.
- иначе → повтори шаг 1 с `--limit 30`. Если и там моих < 3 / < 70 слов — это «новый» канал/контакт; референсы из шага 3 становятся **основным** источником стиля, а не дополнением.

### Шаг 3. Референсы

Просмотри `${CLAUDE_PLUGIN_ROOT}/skills/mm-reply/references.md`:

Каждая строка-ссылка имеет вид:

```
- <permalink> — <одно короткое описание стиля/тона/контекста> + может быть указано, что нужно прочитать root + треды ПОД ним.
```

Найди описания, релевантные **тону** запроса (если задан) или **типу задачи** (отказ / объяснение / уточнение / благодарность). Примеры матчей:
- «ответь холодно» → ищи «холод», «отказ», «коротк», «сухо»
- «мягко объясни» → «мягк», «тёпл», «объясн», «развёрн»
- «строго уточни» → «строг», «без воды», «прямо»

Для каждого матча подтяни тело:

```bash
uv run --no-project python ${CLAUDE_PLUGIN_ROOT}/scripts/mm_fetch.py '<permalink>' # --mode all если указано
```

Лимиты:
- максимум **3 референса** за раз. Если матчей больше — оставь самые точные.

### Шаг 4. Черновик

Сформулируй сообщение, удерживая:

- **стиль и тон** из (a) её сообщений в шаге 1, (b) подгруженных референсов.
- **факты** — только из контекста переписки, ресёрча явного намерения пользователя, и подгруженных референсов. Не выдумывай детали (имена коллег, даты, цифры, ссылки).

### Шаг 5. Чекпоинт — апрув

Покажи пользователю черновик.
Дальше **жди апрув или правки**. 


## Известные сбои

- `HTTP 401 session_expired` → кука протухла. Скрипты сами выкидывают понятный текст. Скажи пользователю: «MMAUTHTOKEN протух, см. `cookie.md` в `~/Documents/auto-agents/mattermost-threads/operate-mm/`». Не пытайся продолжать workflow до обновления `.env`.
- `users/username/X → 404` → переспроси точное написание ника либо попроси прямой URL.
- `references.md` не существует или пуст → пропусти шаг 3, веди только по шагу 1. Скажи пользователю: «референсов нет, опираюсь только на твой стиль в этой переписке».
