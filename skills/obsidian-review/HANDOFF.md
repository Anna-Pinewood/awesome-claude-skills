# HANDOFF — obsidian-review

Обновлено: 2026-07-21, конец сессии разработки минимум-версии.

## Single Grasp

Мы построили систему ревью правок Claude Code в Obsidian: агент правит волт →
изменённые страницы сами открываются с красно-зелёной подсветкой → принять или
отклонить можно отдельное изменение (удаление и добавление — раздельно!), файл
или всё разом. **Минимум-версия готова, работает и подтверждена пользователем
живым прогоном.** Остановились на точке «всё работает, можно жить и пользоваться».
Следующий большой шаг, если захочется продолжать — максимум-версия: комментарии
и вопросы к строкам с прокидкой обратно в Claude Code (нужна отдельная спека).
Чтобы просто продолжить пользоваться — ничего делать не надо, всё уже включено.

## Context

Ольга ведёт второй мозг в Obsidian (`~/Yandex.Disk.localized/obsidian-vault/cloud-base`,
~470 md-файлов, синк Яндекс.Диском, пользуется одна). Агенты Claude Code регулярно
правят волт, и до этой системы правки применялись «вслепую» — не было способа
увидеть, что именно изменилось, и откатить лишнее. Идея выросла из паттерна
`agentic-kit/skills/review` (браузерное ревью), но для Obsidian и с ключевым
требованием: детерминизм через хуки, а не надежда на дисциплину модели.

## Goal

Минимум-версия (готова): подсветка диффов на страницах Obsidian + accept/reject
на трёх уровнях + side-by-side + панель файлов. Спека:
`docs/specs/2026-07-21-obsidian-review-design.md`.

## Progress narrative

Брейнсторм → спека (выбрано: shadow-git вне волта как базлайн; оркестрация
хуками PreToolUse/Stop с файлом-маркером `task-active`; тонкий плагин-рендерер
на `@codemirror/merge`; состояние ревью только в памяти сессии Obsidian).
Реализация скриптов + smoke-тест — быстро и чисто. Плагин — длинная серия
итераций отладки рендеринга, в которой ключевыми оказались четыре бага
(см. «What didn't work»). Для отладки построили себе инфраструктуру прямо в
плагине: `GET /status` (полное самонаблюдение вплоть до computed-стилей и
координат ханков), `POST /reload` (самоперезагрузка), `POST /op` (программное
нажатие кнопок) — после этого отладка шла без участия пользователя. Финал:
раздельные кнопки удаления/добавления на построчном LCS, юнит-тесты, живой
автопрогон всех операций, подтверждение пользователем.

## What worked

- **Хуки вместо скилла-дирижёра** — идея Ольги; агент может вообще не знать о
  системе ревью, правки ловятся детерминированно. Bash-записи ловим эвристикой:
  текст команды упоминает путь/имя волта (кейс `cd` ловится на самом cd).
- **Debug-endpoints в плагине** (/status, /op, /reload) — превратили отладку
  «пришлите скриншот и перезагрузите» в автономный цикл. Оставлены в коде.
- **Чистая математика отдельно + юнит-тесты** (`chunkOps.ts`): все краевые
  случаи (EOF без \n) отловлены bun-тестами за минуты, без Obsidian.
- **Пакет доставки = «путь + старый текст целиком»**: дифф считает
  `@codemirror/merge`, свой формат ханков не нужен.
- **Кумулятивный базлайн**: файл уже в ревью → новый push не затирает старый
  original — подсветка накапливается, ничего не теряется молча.

## What didn't work

- **`syntaxHighlightDeletions` (default true) роняет Obsidian**: подсветка
  синтаксиса удалённых блоков гоняет кастомный маркдаун-парсер Obsidian вне
  редактора → «Cannot read properties of null (reading 'viewport')» → рушится
  ВЕСЬ рендеринг декораций. Обязательно `syntaxHighlightDeletions: false`.
- **Обновление original через reconfigure компартмента — не работает**:
  CodeMirror сохраняет значения StateField при реконфигурации. Только
  `originalDocChangeEffect`/`updateOriginalDoc`.
- **Deferred views (Obsidian ≥1.7)**: фоновые вкладки — заглушки без методов
  MarkdownView; любой обход вкладок обязан проверять `instanceof MarkdownView`,
  иначе исключения (у нас они вешали HTTP-запросы).
- **`cachedRead` сразу после внешней записи** возвращает устаревший кэш —
  в receivePush только `vault.read`.
- **Side-математика ханков без LCS** дублировала/съедала общую строку, которую
  merge-вью втягивает в ханк на EOF без завершающего \n.
- **WeakSet-учёт «подключённости»** редакторов врёт: Obsidian переиспользует
  EditorView и пересоздаёт состояние — детект только по самому состоянию
  (`getChunks(state) !== null`).
- **FileChanged-хук** не решает задачу «ловить любые изменения md»: не видит
  Bash-записи, матчер без глобов, нет pre-состояния.

## Important details and notes

- Порт плагина 3002 (настройка + `OBSREVIEW_PORT`), токен генерирует плагин,
  скрипты читают его сами из `data.json` — руками не копировать.
- Диагностика: `curl -H "Authorization: Bearer $(jq -r .token <vault>/.obsidian/plugins/obsidian-review/data.json)" http://127.0.0.1:3002/status`.
- Пересборка после правок плагина: `scripts/install-plugin.sh` + POST /reload
  (или руками выкл/вкл в Community plugins).
- Смок git-механики: `scripts/smoke-test.sh`; юнит-тесты: `cd plugin && bun test`.
- Тесты исключены из tsc (`tsconfig.exclude`), typecheck: `bunx tsc --noEmit`.
- Frontmatter исключён из диффа (нормализация базлайна на приёме и на лету) —
  иначе плагин `frontmatter-modified-date` шумел бы в каждом ревью.
- Семантика раздельных кнопок: ✓зелёное — узаконить новое (красное остаётся);
  ✕зелёное — убрать новые строки; ✓красное — забыть старое; ✕красное — вернуть
  старые строки (зелёное не трогается). Красный блок после ✕ исчезает — это
  норма (нечего больше удалять), пользователь сперва принял это за баг.
- В волте осталась тестовая заметка `claude-review-test.md` — можно удалить
  (появится в ревью как «удалено», принять всё).
- Debug-пробы в /status (`origTail`, `btnDel` и т.п.) — рабочие леса; можно
  выпилить при желании почистить код, но они дёшевы и полезны.
