# obsidian-review

Ревью правок Claude Code прямо в Obsidian: агент правит волт → изменённые
страницы открываются сами → удалённое красным, добавленное зелёным → принять
или отклонить можно ханк, файл или всё разом. Inline-режим и side-by-side.

Дизайн: `docs/specs/2026-07-21-obsidian-review-design.md`.

## Как это работает

- **PreToolUse-хук** (`scripts/ensure-baseline.sh`): перед первой пишущей
  операцией задачи в волте коммитит «фото до» в shadow-git
  (`~/.claude-obsidian-review.git`, живёт вне волта — Яндекс.Диск его не видит)
  и ставит маркер задачи.
- **Stop-хук** (`scripts/push-review.sh`): в конце хода агента собирает дифф
  против фото и шлёт плагину на `localhost:3002`. При успехе коммитит новый
  базлайн; если Obsidian закрыт — не коммитит, дифф докатится после следующей
  задачи.
- **Плагин** (`plugin/`): принимает пакет, открывает страницы, рендерит дифф
  через `@codemirror/merge`, держит состояние ревью в памяти (закрыла Obsidian —
  непринятое считается принятым, файлы уже на диске, история — в git-архиве).

## Установка

```bash
scripts/setup.sh            # 1. shadow-git + печатает блок хуков
# 2. добавить напечатанный блок hooks в ~/.claude/settings.json
scripts/install-plugin.sh   # 3. собрать и положить плагин в волт
# 4. Obsidian → Settings → Community plugins → включить «Claude Review»
# 5. перезапустить сессии Claude Code (хуки читаются на старте)
```

Токен авторизации плагин генерирует сам; скрипты читают его из
`data.json` плагина — руками ничего копировать не нужно. Порт по умолчанию
3002 (настройка плагина + `OBSREVIEW_PORT` в `scripts/config.sh`).

## Использование

Просто попросить агента что-то поправить в волте. Дальше всё само.

В ревью: у удалённого (красного) и добавленного (зелёного) — **раздельные**
✓/✕ (✕ на красном возвращает старые строки, не трогая новых); «Принять/Отклонить
файл» — кнопки в шапке вкладки; «Принять/Отклонить все» — боковая панель
«Ревью правок» (там же список файлов с +N −M), палитра команд или статус-бар.
Side-by-side — кнопка-колонки в шапке вкладки.

Отладка: `GET /status` (самонаблюдение плагина), `POST /op` (программное
нажатие кнопок), `POST /reload` (перезагрузка плагина) — все с bearer-токеном.
Текущее состояние работ и грабли — в `HANDOFF.md`.

## Тесты

`scripts/smoke-test.sh` — git-механика хуков на одноразовом волте, без
Obsidian. Плагин проверяется руками сквозным сценарием (см. спеку).

## Известные ограничения

См. раздел «Не входит в минимум» в спеке. Главное: подсветка только в
редакторе (не в Reading mode), состояние ревью не переживает перезапуск
Obsidian, ревью с одного устройства.
