# Навык: after-therapy — «разложи» после сессии терапии

**Что делает:** берёт буфер мыслей из последней session-заметки терапии и разносит пункты по родным страницам кластера, обновляет их `#### Single grasp`, чистит буфер. Пишет сам, без согласования каждой правки; протокол записи (подсветка при вставке + открытие после) — см. `../WRITE-PROTOCOL.md`, читай его **перед** правкой.



## Кластер и адреса

- **Буфер-источник:** последняя `0-claude-mirror/projects/personal/psychotherapy/session-YYYY-MM-DD.md`, секция `## Следующая сессия`.
- **`psychotherapy.md`** — `0-claude-mirror/projects/personal/psychotherapy/psychotherapy.md`
  - метрики-состояния → `## Метрики успешности терапии`
  - режимы тренировки / регулярные практики → `## Практики` (создать, если секции нет)
- **`socialize-journey.md`** — `0-claude-mirror/projects/personal/socialize-journey.md`, полевые наблюдения → `## Логи`
- **`To-do list.md`** — в корне волта, дела/идеи.

## Шаги

1. **Найди буфер.** Последняя `session-YYYY-MM-DD.md` = максимальная дата в имени. Прочитай её секцию `## Следующая сессия`. Прочитай также целевые секции назначения (чтобы попасть в их формат): `psychotherapy.md#Метрики успешности терапии`, `socialize-journey.md#Логи`.

2. **Классифицируй каждый пункт буфера** (пользователь часто уже пишет адрес в скобках — уважай его буквально):
   - строка-таблица `| дата | текст |` (полевое наблюдение) → `socialize-journey.md`, строкой в актуальную таблицу под `## Логи`. Одинаковые эпизоды **слить** (дедуп), не задваивать.
   - новая **метрика-состояние** («я могу …», результат, а не режим) → `psychotherapy.md#Метрики успешности терапии`, буллетом в стиле существующих, от первого лица.
   - **режим тренировки / практика** («2 смолл-тока в день») → `psychotherapy.md#Практики`. НЕ в метрики.
   - **дело / идея / проект** (напр. «сделать методику…») → `To-do list.md`.
   - помечено `(в корневую [[psychotherapy]])`, `(→ socialize)` и т.п. → исполни пометку буквально.
   - **адрес непонятен → не угадывай, спроси** пользователя по этому пункту.

3. **Комментарии психолога** (если у пункта есть) — **вариант A**: коммент едет вместе с пунктом в его дом, под-строкой сразу под пунктом:
   `> психолог, <DD-MM-YYYY>: …`

4. **Запиши** (домен-специфика; общий пост-процесс — шаг 6):
   - добавляй в правильную секцию в её формате (таблица / буллет / лог), не ломая существующее;
   - на каждой затронутой странице обнови `#### Single grasp` (5–10 строк: состояние + что изменилось). Если секции ещё нет — создай её наверху, под `# H1`.

5. **Очисти буфер:** убери разложённые пункты из `## Следующая сессия`, оставь пустой заголовок (+ подзаголовок «Мысли», если был) для следующего цикла.

## Заметки
- Даты в тексте — `<DD-MM-YYYY>` (греппабельно, не триггерит вики-линк-парсер).
- Не дублируй: наблюдение уже в логе — слей, не плоди.
- Специфика (страницы, форматы, секции) живёт здесь, в этом листе. Общий протокол записи (подсветка, открытие, снятие подсветки, отчёт) — в `../WRITE-PROTOCOL.md`, вынесен туда при появлении второго листа-редактора (`update-plans`).
