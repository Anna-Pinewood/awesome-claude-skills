# Навык: update-plans — «внеси апдейты» по проектам

**Что делает:** берёт один или несколько апдейтов (короткие новости/решения по текущим делам), для каждого находит целевую страницу через `projects-bucket.md`, дописывает его в подходящую секцию этой страницы её языком, обновляет `#### Single grasp`. Пишет сам, без согласования по каждому апдейту; протокол записи (подсветка при вставке + открытие после) — см. `../WRITE-PROTOCOL.md`, читай его **перед** правкой.

## Источник
Пользователь диктует апдейт(ы) в чате. Апдейтов может быть несколько за раз — обрабатывай каждый отдельно по шагам ниже.

## Шаги

1. **Найди целевую страницу через `projects-bucket.md`** (`0-claude-mirror/projects-bucket.md` — там вся актуальная картина):
   - Не читай файл целиком. Возьми только строки со ссылками: `grep -n '\[\[' projects-bucket.md`.
   - Читай эти строки целиком, **батчами по 10**, сверху вниз. Верх файла — подсвеченное и активное, низ — фон/бакет/«когда-нибудь»: совпадение почти всегда находится в первых батчах, читать до конца обычно не нужно.
   - Как только нашлась страница, тема которой явно соответствует апдейту — бери её и дальше не читай.
   - Ничего не подошло, или несколько кандидатов одинаково похожи → спроси пользователя, в какую страницу писать.

2. **Найди нужную секцию через заголовки**, не читай страницу целиком:
   - Оглавление на лету: `grep -n '^#' <страница>.md` (страница не хранит оглавление, строим на лету).
   - Определи по заголовкам, в какую секцию писать.
   - **Прочитай саму секцию** — это нужно, чтобы убедиться и понять формат для встраивания (буллеты / таблица / нумерация и т.п.).
   - Несколько заголовков одинаково подходят или ни один не подошёл → спроси пользователя, в какую секцию писать.
   - Пиши **максимально языком пользователя** — сохраняй её формулировку и интонацию (не добавляй свой синтез, не превращай мысль в вывод, которого не было; можно поправить пунктуацию, слить повторы, добавить структуру, если применимо, не больше).

3. **Обнови `#### Single grasp`** страницы, если она есть — коротко впиши, что изменилось. Если секции ещё нет — не создавай её здесь специально (это забота листа, который вводил страницу).

## Заметки
- Несколько апдейтов за раз — у каждого свой адрес; одна и та же страница может встретиться дважды.
- Не дублируй: если апдейт уже отражён на странице — слей с существующим, не плоди.
